# 一、多媒体系统简介
按照CCITT(国际电话电报质询委员会)多媒体分为下列几类

1. 感觉媒体(Perception Medium)：  **直接作用于人的感觉器官** ，使人产生直接感觉的媒体。例如听觉反应的声音、引起视觉反应得图像。

2. 表示媒体(Representation Medium)：传输感觉媒体得中介媒体， **用于数据交换编码**。例如，图像编码(JEPG、MPEG)、文本编码(ASCII、GB2312)、声音编码等

3. 表现媒体(Presentation Medium)： **进行信息输入和输出得媒体** 。例如，键盘、鼠标、扫描仪、话筒、摄像机等为输入设备；显示器、打印机、喇叭等为输出设备

4. 交换媒体(Interchange Medium)： **系统之间进行数据交换得媒体** 。包括存储媒体和传输媒体。
    * 存储媒体：存储表示媒体得物理介质，例如，硬盘、软盘、磁盘、光盘、ROM及RAM等
    * 传输媒体: 传输表示媒体得物理介质，例如，电缆、光缆、电磁波等

多媒体技术使指利用计算机技术把 **文本、图形、图像、声音、动画和电视** 等多种媒体综合起来，使多种信息 **建立逻辑连接** ，并能够对它们进行 **获取、压缩、加工处理、存储** ，集成为一个具有 **交互性的系统**。

为了加快多媒体信息处理速度，可以为通用CPU增加特殊的多媒体处理指令，也可以设计专用的协处理器来处理多媒体信息。另一方面。可以通过系统总线扩展，接入各类板卡；例如，音频卡、视频采集卡、图形图像加速卡等。集成以PC为中心的组合平台。

## 1.1 数字声音

声音信号的两个基本参数使振幅和频率。振幅一般用分贝(dB)为单位计量，频率指声波每秒变化频率，以Hz表示

人耳能够听见的声音频率是20Hz~20000Hz. 对于小于20Hz称为次声波，高于20000Hz的称为超声波。

* 数字话音信号的频率定义在300~3400Hz。
* CD数字声音的频率定义在20Hz~20000Hz.

根据声音的频率的构成特征可以将声音分为乐音和噪音两种信号。

* 基音频率：物体振动发出的具有清晰可辨的音高；其他频率成分称为泛音
    * 泛音的频率是基因频率的整数倍，称这个复合音位乐音。

* 噪音：**非整数倍基音频率的泛音** ，这个声音是不具备清晰可辨的音高

* 音色：由基音与泛音的比例、泛音分布、泛音随时间的衰变决定
 
### 1.1.1 声音信号的数字化
声音信号是一种模拟信号，将模拟信号数字化的过程为
1. **采样**。 把时间连续的模拟信号离散化的过程。**在某一特定时刻(相等的时间间隔)获取声音信号的振幅**。两次采样的时间间隔称为采样周期，采样周期的倒数称为采样频率。 **采样频率需要大于声音信号最高频率的两倍** 

2. **量化**。把在振幅上连续取值(模拟量)的每一个样本转换为离散值(数字量)表示。即 **对样本振幅值进行A/D转换(模数转换)** ，量化后的样本用二进制表。
    * 量化精度，二进制数的位数反映了度量声音波形幅度的精度，也称量化分辨率。量化精度越高，声音的重构质量越好，需要的存储空间越多

3. **编码**。经过采样和量化后的声音采样信号变成了数字形式了，为便于存储、处理和传输，还需要进行数据编码，按照某种规定的格式将数据组织成文件。

经过数字化处理后的数字声音的主要参数

|参数|说明|附加|
|:---:|:---:|:---:|
|采样频率|每秒内采样次数|标准频率为`44.1Hz、22.05Hz、11.05Hz`|
|量化位数|度量声音波形幅度的精度|一般为`8bit、12bit、16bit`|
|声道数目|单声道一次产生一组声音波形数据，双声道一次产生两组声音波形数据||
|数据率|每秒钟的数据量|以bps为单位|
|压缩比|单位时间内的未压缩音频数据率与压缩后的数据量之比||


### 1.1.2 声音的表示

计算机中声音的两种表示方法：波形声音信息和非波形描述声音的声音信息。
* 波形声音信息: 对实际声音波形信号进行数字化(采样和量化)获得，能够真实地表示现实世界中任何客观存在的声音。数据量一般比较大
* 非波形描述的声音信号：使用符号(参数)、脚本、模型等对声音进行描述，通过计算机合成的方法重构声音信号。数据量小，但是声音没有自然声真实。

声音信息处理技术包括**声音获取、重建及播放；数字声音的编辑处理；存储与检索；传输；与文本的相互转换等**。

波形声音信息是对声音波形的直接采样数据，未经过压缩的数字音频传输率(bps)为$数据传输率(bps)=采样频率(Hz)\times 量化位数(bit)\times 声道数$；占用的存储空间为$声音信号数据量(Byte)=数据传输率\times \frac{持续时间(s)}{8}$

### 1.1.3 声音合成
计算机和多媒体系统中的声音，处理数字波形声音外，还有一类是使用符号参数来表示的。如果要播放出来，则需要由计算机来合成出声音信号，包括语音合成和音乐合成。例如, MIDI格式的数字音乐

### 1.1.4 MIDI
MIDI(Musical Instrument Digital Interface)乐器数字接口国际标准。它规定了电子乐器与计算机之间、电子乐器之间硬件互联及数据通信协议的完整规范。该标准定义了计算机音乐程序、合成器及其他电子设备交换信息和电子信号的方式，解决了不同电子乐器之间的兼容性问题。

对于符合该标准的设备称为MIDI设备。通过MIDI接口可以解决不同MIDI设备之间的信息交互问题。

计算机音乐系统是由带有MIDI接口及专用MIDI电缆计算机、合成器和其他MIDI设备连接在一起构成。

数据由MIDI设备的键盘产生，通过声音合成器还原为声音，计算机可以控制乐器的输出，并能接收、存储和处理经过编码的音乐数据。

MIDI文件时计算机中用于存储和交换MIDI消息的数据文件，包含了乐曲演奏过程的完整信息。标准MIDI文件格式采用`.mid`为文件扩展名，它是音乐软件的文件交换标准，也是商业电子音乐作品发行的标准。

MIDI音乐信息的音质取决于音乐合成器和音源硬件的波标音色。


### 1.1.5 声音文件格式
数字声音在计算机中存储和处理时，其数据必须以文件的形式进行组织，所选用的文件格式必须得到操作系统和应用软件的支持。常见的声音文件格式如下
1. Wave文件(`.wav`)。Windows系统中使用的标准音频文件格式， **源于声音波形采样** 。该格式的声音文件能够和原声基本一致，质量非常高，但是文件是数据大。
2. Sound文件(`.snd`)。NeXT Computer公司推出的数字声音文件格式，支持压缩
3. Audio文件(`.au`)。(Sun Microsystems公司推出)广泛应用于UNIX系统中的数字声音问价格式
4. AIFF文件(`.aif`)。Mac os的标准音频文件格式
5. Voice文件(`.voc`)。(Creative公司)波形音频问价格式。每个VOC文件由文件头块和音频数据块组成。文件头包含一个标识版本号和一个指向数据块起始的指针。
6. MPEG-1 Audio Layer 3文件(`.mp3`)。最流行的音频文件格式，压缩率大，能够以极低的码率提供接近CD音质的声音效果
7. RealAudio文件(`.ra`)。具有强大的压缩量和极小的失真，为了解决网络传输带宽不足而设计，主要解决压缩比和容错性
8. MIDI文件(`.mid/.rmi`)。目前较成熟的非波形采样点音乐格式。MIDI文件包含的时音乐演奏指令，文件长度小。`.RMI`还可以包含图片标记和文本
